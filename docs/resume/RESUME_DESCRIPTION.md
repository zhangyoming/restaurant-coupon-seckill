# 简历描述参考：餐饮门店优惠券秒杀系统

## 1. 项目名称

餐饮门店优惠券秒杀系统

## 2. 项目描述

基于 Spring Boot、Redis、Lua、RabbitMQ 和 MySQL 实现的餐饮门店优惠券秒杀系统，面向门店限量优惠券抢购场景，支持优惠券活动创建、活动发布、库存预热、验证码、秒杀令牌、用户抢券、异步发券、结果查询、接口限流和 JMeter 压测，解决高并发场景下的超卖、重复领取和数据库压力问题。

## 3. 技术栈

Java 17、Spring Boot、MyBatis Plus、MySQL、Redis、Lua、RabbitMQ、Knife4j、JMeter、Docker

## 4. 简历要点

可以从下面选择 5 到 7 条写到简历里：

```text
1. 设计优惠券秒杀活动、领取记录和秒杀订单模型，基于 user_id + activity_id 唯一索引防止同一用户重复领取。

2. 活动发布后将库存预热到 Redis，抢券请求优先访问 Redis 库存，减少高并发场景下对 MySQL 的直接访问。

3. 使用 Redis Lua 脚本原子完成库存校验、重复领取校验、库存扣减和用户领取标记，避免多 Redis 命令并发执行导致超卖和一人多领。

4. 使用 RabbitMQ 实现异步发券，抢券接口只创建排队订单并投递消息，消费者异步扣减数据库库存和创建领取记录，实现削峰填谷。

5. 针对 MQ 重复消费场景，基于秒杀订单状态实现幂等控制，并通过数据库唯一索引作为重复发券兜底。

6. 设计 Redis 预扣库存补偿机制，当数据库落库失败时回滚 Redis 已领取用户标记并恢复库存，保证缓存和数据库最终一致。

7. 实现验证码、短期一次性秒杀令牌、用户维度限流和 IP 维度限流，降低机器人刷接口和活动未开始前提前请求风险。

8. 使用 JMeter 设计核心抢券链路和完整防刷链路压测脚本，并通过订单表、领取记录表、活动库存校验高并发下不超卖、不重复发券。
```

## 5. 面试讲法

项目可以按这条主线讲：

```text
我先用数据库唯一索引和条件扣库存实现基础版抢券，保证功能正确；
然后把活动库存预热到 Redis，减少数据库访问；
接着用 Redis Lua 原子完成库存扣减和重复领取判断，解决 Redis 多命令并发问题；
再使用 RabbitMQ 做异步发券，接口立即返回排队中，由消费者异步落库；
最后增加验证码、秒杀令牌、用户/IP 限流和 JMeter 压测，验证高并发下不超卖、不重复发券。
```

## 6. 不建议这样写

不要只写：

```text
使用 Redis、RabbitMQ、JMeter 实现秒杀系统。
```

这种写法太空。

应该写成：

```text
使用 Redis Lua 原子完成库存校验、重复领取校验和库存扣减，使用 RabbitMQ 异步发券削峰填谷，并通过订单状态幂等和唯一索引防止重复发券。
```
